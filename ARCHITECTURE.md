## System Overview

Privacy-first, voice-first React Native app. All ML inference on-device via ONNX Runtime.
No user data leaves the device. Offline-first.

## Module Boundaries

- `screens/` — layout and navigation only. No business logic. No DB access.
- `features/*/` — feature logic, hooks, local state, DB access (via habitDb.ts only)
- `shared/` — pure TypeScript utilities. No React. No state.
- `services/` — platform API wrappers (expo-speech, expo-av, expo-notifications)
- `models/` — ONNX inference wrappers only

## Data Flow

Voice input → ASR (whisper-tiny.onnx) → Language detect → Intent classify → Entity extract → Confidence gate → Habit store (SQLite) → Zustand → UI

## Key Decisions

- React Native over Flutter: ONNX Runtime React Native has first-class JSI support; Flutter bindings less mature
- Zustand over Redux: simpler API, smaller bundle, sufficient for this scale
- Bun over npm/yarn: faster installs on Apple Silicon, native ARM64 test runner
- expo-sqlite over WatermelonDB: official Expo support, simpler integration with new arch
